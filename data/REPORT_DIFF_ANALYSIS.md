# Сравнение output_report.xlsx и expected_report.xlsx

## Метод сравнения

Скрипт `scripts/compare_reports.py`: открытие обоих файлов через openpyxl (data_only=True), сравнение имён и порядка листов, заголовков, числа строк, примеров данных, ширин колонок и жирности шапки.

**Важно:** в expected_report первая строка данных может относиться к другому входному набору (другой кабинет/регион), поэтому расхождения по значениям в ячейках частично связаны с разными исходными данными, а не только с логикой.

## 2. Разделение расхождений по типам

### Структурные

- **Нет.** Состав листов и их порядок совпадают. Набор и порядок колонок по смыслу те же; отличия только в написании заголовков (пробелы).

### Логические

1. **Кабинет Авито** — в output подставлена константа «Кабинет Авито», в expected — реальное имя кабинета («Акрон Услуги»). Нужно брать значение из параметра/конфига, а не дефолтный текст.
2. **Номер на Авито** — у нас выводится ID объявления; в expected — телефон. По output_structure.md в эталоне в этой колонке телефон. Нужно решить: выводить телефон (если есть поле в источнике) или оставить ID; при желании приблизить к expected — маппинг «Номер на Авито» на поле с телефоном.
3. **Куда переадресация** — у нас пусто (redirect_mapping не заполнен или не совпадает с employee). В expected — «Акрона». Нужно заполнить/поправить redirect_mapping.yaml и убедиться, что подставляется значение по сотруднику.
4. **Сводная: число строк и период** — у нас одна строка и пустой «Период»; в expected две строки и заполненный «Период» (формат «ДД.ММ - ДД.ММ»). Нужно: либо один период по датам из файла, либо несколько периодов; заполнять колонку «Период» в том же формате.
5. **Диаграмма / Недельный: разное число строк (160 vs 175 и 160 vs 164)** — уточнить, ожидается ли один и тот же набор строк с expected или допустимо разное (разные входные данные). При необходимости — одна и та же фильтрация/источник строк для всех листов.

### Визуальные

1. **Ширины колонок** — в output не заданы, в expected заданы по каждому листу. Нужно выставить ширины из конфига/эталона (как в expected) в formatter.
2. **Заголовки с неразрывными пробелами** — на листах Недельный и Сводная в expected заголовки содержат `\xa0`. Чтобы визуально и побайтово приблизить к expected, выводить те же заголовки (с `\xa0`) в workbook_layout или в коде формирования заголовков.
3. **Автофильтр на Диаграмме** — в expected включён (A1:E161). У нас уже может быть из layout; проверить, что диапазон совпадает с фактическим (например, по числу строк).

## 3. Список правок по приоритету

### Высокий приоритет (логика и консистентность)

1. **Кабинет Авито** — подставлять в листы значение из параметра `cabinet_name` (или конфиг), а не строку «Кабинет Авито».
2. **Куда переадресация** — заполнить `redirect_mapping.yaml` по фактическим сотрудникам и убедиться, что в отчёте подставляется значение (как «Акрона» в expected).
3. **Сводная: период** — заполнять колонку «Период» (и при необходимости «Неделя»), формат как в expected («16.02 - 22.02»); при необходимости — несколько строк за несколько периодов.

### Средний приоритет (соответствие эталону и данным)

4. **Номер на Авито** — уточнить ТЗ: выводить телефон или ID; если телефон — добавить маппинг из входного поля (если оно есть) в колонку «Номер на Авито».
5. **Сводная: число строк** — при необходимости два периода — считать/формировать две строки сводной (по периодам), как в expected.
6. **Диаграмма: автофильтр** — проверить, что автофильтр на Диаграмме включён и диапазон соответствует данным (аналог A1:E161 с актуальным количеством строк).

### Низкий приоритет (оформление)

7. **Ширины колонок** — задать в workbook_layout или formatter ширины по листам (взять значения из expected) и применять при записи.
8. **Неразрывные пробелы в заголовках** — для листов Недельный и Сводная выводить заголовки с `\xa0`, как в expected, если нужна максимальная близость к эталону.

## 4. Что не менять (или уточнить по ТЗ)

- **Область** — в expected местами пусто; у нас заполнено. Оба варианта допустимы по output_structure.md; менять логику только при явном требовании.
- **Разное число строк (160 vs 175, 160 vs 164)** — может быть следствием другого входного файла у expected; при необходимости зафиксировать в ТЗ, что ожидается одно и то же количество строк при одном и том же входе.

Код пока не менялся; следующий шаг — реализовать правки по этому списку в коде (в нужном порядке приоритетов).
