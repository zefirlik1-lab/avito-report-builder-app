## 1.6. Рекомендации для реализации pipeline

### Нормализация заголовков
Рекомендуется:
- заменить неразрывные пробелы на обычные;
- схлопывать лишние пробелы;
- приводить заголовки к точному сопоставлению с каноническими именами.

### Нормализация текстовых полей
Рекомендуется:
- обрезать пробелы по краям;
- заменить неразрывные пробелы;
- нормализовать повторные пробелы;
- приводить пустые строки к `None` / пустому значению внутренней модели.

### Обработка `HYPERLINK(...)`
Поскольку:
- `Номер объявления` приходит как `HYPERLINK(...)`,
- `Название объявления` тоже приходит как `HYPERLINK(...)`,

рекомендуется заранее определить модель хранения.

Практичный вариант:
- для номера объявления хранить:
  - отображаемый текст,
  - URL отдельно;
- для названия объявления хранить:
  - отображаемый текст,
  - URL отдельно.

То есть логически подготовить возможность извлечения:
- текста,
- ссылки,
- либо обоих значений.

### Обработка поля `Цена`
Поскольку `Цена` визуально представлена как строка вида `5 000 ₽`, рекомендуется:
- удалить символ валюты;
- удалить обычные и неразрывные пробелы;
- затем приводить значение к числу только в том случае, если поле реально потребуется в бизнес-логике.

### Обработка дат
Поскольку даты визуально понятны, но формат ячеек — `General`, рекомендуется:
- выполнять явную нормализацию даты при чтении;
- не полагаться на автоматическое распознавание Excel.

### Обработка процентов
Поскольку процентные поля визуально отображаются как проценты, но по данным выглядят как числовые доли, рекомендуется:
- хранить их как numeric / float;
- не выполнять повторное деление на 100.

### Обработка денежных полей
Рекомендуется:
- читать денежные поля как numeric / float;
- при необходимости применять округление только на уровне формирования итогового Excel, а не на уровне сырой модели данных.

### Работа с полями, не использующимися напрямую в шаблоне
Даже если часть полей не видна в текущем шаблоне, рекомендуется:
- не удалять их из исходного pipeline слишком рано;
- сначала сохранить их во внутренней нормализованной модели;
- исключать только на этапе построения конкретных выходных листов, если не подтверждена их ненужность.
