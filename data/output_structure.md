## 2.3. Лист `Админ панель`

### Факты, наблюдаемые в Excel

#### Предполагаемое назначение
Справочный лист по объявлениям. Содержит кабинет, название объявления, географию, адрес, номер, переадресацию и ID объявления.

#### Размер
- 161 строка
- 8 колонок

#### Колонки по порядку
1. `Кабинет Авито`
2. `Название`
3. `Область`
4. `Город`
5. `Адрес`
6. `Номер на Авито`
7. `Куда переадресация`
8. `ID объявления`

#### Наблюдаемая структура данных
- `Название` заполнено формулами `HYPERLINK(...)`
- `ID объявления` заполнено формулами `HYPERLINK(...)`
- `Область` местами пустая
- `Номер на Авито` в примере выглядит как телефонный номер, а не ID объявления

#### Важные особенности форматирования
- объединений ячеек нет
- автофильтр не включён
- закрепления строк нет
- заголовок жирный
- заголовки выровнены в основном по центру
- ширины столбцов выставлены вручную

#### Формулы
Есть формулы `HYPERLINK(...)`:
- в колонке `Название`
- в колонке `ID объявления`

Других формул по наблюдению нет.

### Рекомендации для реализации pipeline
- Формировать лист как отдельное табличное представление, а не как побочный результат другого листа.
- Поддержать вывод гиперссылок как минимум в колонках:
  - `Название`
  - `ID объявления`
- Поле `Номер на Авито` не следует автоматически трактовать как ID без отдельного подтверждения, так как в примере оно визуально похоже на телефонный номер.
- Поле `Область` следует считать допускающим пустые значения.
- Поскольку автофильтр и закрепление строк здесь не наблюдаются, не следует добавлять их автоматически без отдельного согласования.

## 2.4. Лист `Диаграмма`

### Факты, наблюдаемые в Excel

#### Предполагаемое назначение
Упрощённый лист со справочной структурой для визуализации, аналитики или связки с диаграммами.

#### Размер
- 176 строк
- 5 колонок

#### Колонки по порядку
1. `Кабинет Авито`
2. `Тип`
3. `Название`
4. `Адрес`
5. `ID объявления`

#### Наблюдаемая структура данных
- `Тип` содержит значения вроде `Общее`, `АКБ`
- `Название` заполнено формулами `HYPERLINK(...)`
- `ID объявления` заполнено формулами `HYPERLINK(...)`

#### Важные особенности форматирования
- включён автофильтр: `A1:E161`
- объединений ячеек нет
- закрепления строк нет
- заголовок жирный
- ширины колонок настроены вручную

#### Формулы
Есть формулы `HYPERLINK(...)`:
- в колонке `Название`
- в колонке `ID объявления`

Других формул по наблюдению нет.

#### Что нельзя определить точно
Нельзя точно определить, почему на листе 176 строк при диапазоне автофильтра до 161 строки. Возможные причины:
- ниже есть дополнительные строки, не входящие в автофильтр;
- файл изменялся вручную;
- шаблон заполнялся частично.

### Рекомендации для реализации pipeline
- Поле `Тип` следует считать вычисляемым и формируемым отдельным правилом, а не прямой копией входной колонки.
- Гиперссылки следует поддержать в колонках:
  - `Название`
  - `ID объявления`
- Автофильтр для этого листа следует считать частью текущего шаблона.
- Диапазон автофильтра лучше задавать динамически по фактическому числу строк, а не жёстко повторять наблюдаемый диапазон, если нет отдельного требования копировать шаблон побайтно.

## 2.6. Лист `Сводная`

### Факты, наблюдаемые в Excel

#### Предполагаемое назначение
Сводка по периодам с итоговыми метриками и текстовым комментарием.

#### Размер
- 3 строки
- 10 колонок

#### Колонки по порядку
1. `Неделя`
2. `Период`
3. `Просмотры`
4. `Конверсия из просмотров в контакты`
5. `Контакты`
6. `Средняя цена контакта`
7. `Звонки`
8. `Написали в чат`
9. `Расходы на объявления`
10. `Комментарий`

#### Наблюдаемая структура данных
- в примере 2 строки данных
- поле `Неделя` пустое в обеих строках
- поле `Период` заполнено строками вида `16.02 - 22.02`
- `Комментарий` заполнен обычным текстом вручную
- формул на листе не обнаружено

#### Важные особенности форматирования
- объединений ячеек нет
- автофильтр не включён
- закрепления строк нет
- заголовок жирный
- ширины колонок настроены вручную
- колонка `Комментарий` заметно шире остальных

#### Формулы
Формул на листе не обнаружено.

#### Что нельзя определить точно
По готовому файлу нельзя точно определить:
- рассчитывались ли значения в коде до выгрузки;
- или ранее использовались Excel-формулы, а потом были заменены значениями.

В присланном файле на листе `Сводная` формул нет.

### Рекомендации для реализации pipeline
- Лист следует формировать как агрегированное представление по периоду.
- В текущем шаблоне значения на листе выглядят как готовые данные, а не как формулы, поэтому безопаснее считать основные агрегаты в pipeline, а не полагаться на Excel-формулы.
- Поле `Комментарий` следует считать редактируемым человеком после генерации и не пытаться автоматически заполнять без отдельного правила.
- Поле `Неделя` может быть пустым в текущем шаблоне; не следует делать его обязательным без отдельного подтверждения.
- Поле `Период` следует формировать в строковом виде, совместимом с шаблоном наподобие `16.02 - 22.02`.

## 2.7. Общие особенности итогового файла

### Факты, наблюдаемые в Excel
- объединений ячеек нет ни на одном листе;
- заголовки на всех листах жирные;
- листы оформлены как обычные диапазоны, а не как Excel Table;
- ширины столбцов везде выставлены вручную;
- в данных активно используются `HYPERLINK(...)`;
- сложного декоративного оформления не наблюдается.

### Рекомендации для реализации pipeline
- Сохранять порядок листов:
  1. `Админ панель`
  2. `Диаграмма`
  3. `Недельный`
  4. `Сводная`
- Сохранять названия листов без изменений.
- Сохранять порядок колонок на каждом листе как в шаблоне.
- Поддержать гиперссылки в тех колонках, где они есть в шаблоне.
- Сохранять базовую читаемость:
  - жирные заголовки,
  - настроенные ширины столбцов,
  - адекватные числовые форматы.
- Не добавлять объединения ячеек, если этого нет в шаблоне.
- Не превращать диапазоны в Excel Table без отдельного решения.
- Автофильтры следует настраивать только там, где они реально предусмотрены текущим шаблоном.

# 3. Что нельзя определить точно только по двум файлам

Ниже перечислены вопросы, которые не подтверждаются на 100% только анализом двух Excel-файлов и требуют отдельной фиксации в требованиях:

- какие поля во входном файле считаются строго обязательными бизнес-логикой, а какие просто присутствуют в примере;
- полный набор правил, по которым вычисляется поле `Тип`;
- откуда в точности берётся `Кабинет Авито`;
- из какого точного поля формируется `Куда переадресация`;
- должен ли `Номер на Авито` быть телефоном, ID или иным значением;
- нужно ли воспроизводить оформление шаблона строго визуально, либо достаточно сохранить структуру и читаемость;
- должны ли некоторые значения в итоговом файле рассчитываться формулами Excel, либо они всегда должны записываться как готовые значения.
